# 🌐 Web界面使用指南

## 简介

现在你可以通过浏览器方便地使用VC追踪器了！只需点击按钮，即可抓取并查看来自4个顶级数据源的创业投资资讯。

---

## 🚀 快速开始

### 方法一：使用启动脚本（推荐）

```bash
# 进入项目目录
cd "News-about-VC"

# 运行启动脚本
./start_web.sh
```

### 方法二：手动启动

```bash
# 进入项目目录
cd "News-about-VC"

# 激活虚拟环境（如果存在）
source venv/bin/activate

# 启动Web服务器
python web_server.py
```

---

## 📱 访问Web界面

启动后，浏览器会自动打开，或者手动访问：

```
http://localhost:8080
```

## 🚨 重要提示：数据源限制

### 可稳定抓取的数据源 ✅

1. **Hacker News** - 全球技术创业热点，更新频繁
2. **Paul Graham** - 创业投资经典文章，质量极高

### 受限的数据源 ⚠️

3. **Product Hunt** - 使用JavaScript动态渲染，建议使用RSS或浏览器直接访问
4. **IT桔子** - 需要登录，有反爬虫机制，建议直接访问网站

当抓取受限网站失败时，界面会显示提示信息，您可以点击链接直接访问网站。

**详情请查看**: `DATA_SOURCE_LIMITS.md`

## ✨ 界面功能

### 1. 头部区域
- **标题**: VC追踪器 v2.0
- **数据源徽章**: 显示支持的4个数据来源

### 2. 控制按钮
- **🔄 立即抓取**: 点击开始从4个数据源抓取最新资讯
- **📂 加载已有数据**: 加载之前保存的数据（不重新抓取）

### 3. 筛选功能
- **来源筛选**: 按数据源筛选（Paul Graham / Hacker News / Product Hunt / IT桔子）
- **搜索框**: 输入关键词搜索标题和摘要

### 4. 统计面板
实时显示各数据源的新闻数量：
- 总新闻数
- Paul Graham 文章数
- Hacker News 文章数
- Product Hunt 产品数
- IT桔子 投资事件数
- 最后更新时间

### 5. 新闻列表
以表格形式展示所有新闻，包含：
- **序号**: 编号
- **来源**: 用颜色标签标识数据源
- **标题/摘要**: 可点击跳转到原文，带摘要预览
- **领域**: 自动识别的赛道/行业
- **发布时间**: 格式化的时间显示

## 📋 使用流程

### 第一次使用

1. **启动Web服务器**
   ```bash
   ./start_web.sh
   ```

2. **在浏览器中点击"立即抓取"**
   - 等待抓取完成（约1-2分钟）
   - 会显示加载动画
   - 完成后会提示抓取结果

3. **浏览新闻列表**
   - 点击标题可跳转到原文
   - 使用筛选和搜索功能
   - 查看各数据源统计

### 日常使用

1. **快速查看最新资讯**
   ```bash
   # 启动服务器
   ./start_web.sh
   
   # 浏览器会自动打开
   # 点击"立即抓取"获取最新数据
   ```

2. **查看已有数据**
   - 如果不需要最新数据，点击"加载已有数据"
   - 会显示上次抓取的数据

## 🔧 高级功能

### 数据保留策略
- 数据自动保存到 `data/multi_source_news.json`
- 只保留最近90天的数据，自动清理过期内容
- 数据会持久化，重启服务器后仍可查看

### API接口

如果需要集成到其他系统，可以使用以下API：

#### 获取数据
```http
GET http://localhost:8080/api/data
```
返回JSON格式的所有新闻数据。

#### 刷新数据
```http
POST http://localhost:8080/api/refresh
```
触发重新抓取，返回抓取结果。

## ⚠️ 注意事项

1. **网络要求**
   - 需要能访问外网（Paul Graham、HN、Product Hunt）
   - IT桔子在国内访问更快

2. **抓取时间**
   - 首次抓取约需1-2分钟
   - 取决于网络状况和网站响应速度
   - 请耐心等待，不要重复点击

3. **浏览器兼容性**
   - 支持现代浏览器（Chrome、Firefox、Safari、Edge）
   - 不支持IE浏览器

4. **停止服务器**
   - 按 `Ctrl+C` 即可停止Web服务器
   - 数据会自动保存，不会丢失

## 🐛 故障排除

### 问题1：无法启动服务器
**解决**: 检查端口8080是否被占用
```bash
# 查看端口占用
lsof -i :8080

# 如果被占用，可以修改web_server.py中的PORT变量
```

### 问题2：抓取失败
**解决**: 检查网络连接
```bash
# 测试网络
curl https://news.ycombinator.com

# 如果超时，可能需要配置代理
```

### 问题3：页面显示空白
**解决**: 清除浏览器缓存或刷新页面
- 按 `Ctrl+Shift+R` 强制刷新
- 或清除浏览器缓存后重试

### 问题4：数据不更新
**解决**: 检查数据文件权限
```bash
# 确保data目录可写
chmod 755 data
```
